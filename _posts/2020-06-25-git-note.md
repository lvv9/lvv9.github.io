# Git笔记

0. PRE<br>
工作区（working tree）——暂存区（index/stage）——仓库（repository）
![git基础概念](https://github.com/lvv9/lvv9.github.io/blob/master/pic/1352126739_7909.jpg?raw=true)

1. 创建仓库<br>
git init<br>
会根据当前的目录创建一个本地仓库<br>
git remote add<br>
创建仓库后可以关联空的远程仓库<br>
git push -u<br>
将本地创建的仓库推送至远程仓库，u即upstream，在没关联远程仓库的分支时可以用这个

2. 克隆远程仓库<br>
git clone

3. 工作区提交至暂存区<br>
git add \<filename><br>
git rm \<filename><br>
暂存区增加、删除，命令后面加的是工作区的文件路径<br>
git status<br>
查看状态

4. 暂存区提交至仓库<br>
git commit<br>
git status<br>
查看状态

5. 撤销工作区修改<br>
git checkout -- \<filename>

6. 撤销暂存区修改
git reset HEAD

7. 撤销本地仓库修改<br>
git reset --hard<br>
git log<br>
查看提交日志<br>
git reflog<br>
查看命令日志

8. 创建分支<br>
git branch dev

9. 切换分支<br>
git switch dev（git checkout dev）<br>
git switch -c dev（git checkout -b dev）<br>
创建并切换分支<br>
git stash<br>
工作区和暂存区是所有分支共享，切换前可以先保存这个分支的现场<br>
git stash list<br>
查看<br>
git stash apply<br>
恢复<br>
git stash drop<br>
删除<br>
git stash pop  
恢复并删除

10. 查看分支<br>
git branch

11. 合并分支<br>
git merge dev<br>
git merge --no-ff dev<br>
忽略ff模式(若分支没有冲突，默认启动）

12. 删除分支<br>
git branch -d dev<br>
git branch -D dev<br>
删除未合并的分支

13. 按提交追平<br>
git cherry-pick

14. 抓取远程分支<br>
git pull origin/dev dev

15. 推送分支<br>
git push origin dev
